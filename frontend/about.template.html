The interface

One self-contained HTML file. No framework, no CDN, no build chain on the front end — the display typeface ships inside the file as a data URI and the whole thing works from a file:// URL. The 3D is plain CSS: a perspective pitch you look across at a low camera angle, player pins standing on it like billboards, and cards that flip in real 3D space.

Lineups

Eight teams' latest verified starting XIs laid out on the pitch by formation. Stars glow gold — tap any player for their card.

Player cards

The player's portrait, tournament goals, assists and minutes on the front; their story and a clickable stats source on the back.

Bracket

Round of 32 through the final at MetLife — winners in gold, shootouts annotated, upcoming fixtures dashed.

Golden Boot

The full scorer table with verified assist counts where they exist, and an honest dash where they don't.

How it stays current

Serper scraper MCP google_search · scrape · batch_scrape

Each researcher gets its own copy of the search-and-scrape server. Cross-checking two sources costs one batch call.

▼

Researcher agents bracket · stats · squads

Fresh pydantic-ai agents per run, on an OpenAI-compatible endpoint (kimi-k2.5 via OpenRouter by default). They read the store, hunt the delta, and report what they can prove.

▼

Datastore MCP the only write path

Reports stage against a validated schema; a commit merges atomically under a file lock, re-validates the whole document, and returns a diff.

▼

worldcup.json → one HTML file

Every commit rebuilds the site: data injected straight into the template, published as a single static file.

Rules the tools enforce

  • Every write needs a source URL. A stat the agent can't cite is a stat that doesn't get written.
  • Settled results are protected. Overwriting a played match requires an explicit correction flag and a note citing the discrepancy.
  • Goal tallies never decrease. A lower number means the source is stale, and the tool says so.
  • Errors point forward. Every rejection names the constraint and the next call to make — agents branch from errors, they don't learn from them.
  • No fallbacks. A failed commit writes nothing and says why. Unverifiable stats render as a dash, not a guess.
